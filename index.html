Features

A Gap-First Research Workflow

Instead of simply asking "what has been researched?", LitFlow AI helps you answer "what should be researched next?"

No Keys Needed

All AI stages run through the secure Host API anna.llm.complete. No API keys are stored — completions run on your signed-in Anna account model.

Graceful Fallbacks

Source discovery fetches papers directly via CORS-enabled CrossRef and falls back gracefully if arXiv request boundaries are throttled.

Structured Syntheses

Generates clean, citation-grounded literature reviews in Markdown, ready to export or copy into your academic documents.

Process

The 5-Stage Literature-Review Engine

LitFlow AI orchestrates a sequential state-machine pipeline directly within a secure, responsive iframe in your Anna workspace.

1

Research Planning

Expands your initial topic into primary research questions, keywords, and a targeted search strategy.

2

Source Discovery

Pulls metadata, abstracts, and citations from CrossRef and arXiv directly from the app window.

3

Insight Extraction

Leverages batched LLM calls to extract methodologies, key findings, and limitations for each paper.

4

Gap Discovery

Clusters the corpus into thematic groups and extracts over-researched, under-explored, and contradictory areas.

5

Literature Review

Synthesizes findings into a final, structured literature review, linked to inline references.

Using LitFlow AI in Anna AI Platform

LitFlow AI is packaged as a native Anna App. To run or install the application as a public user:

  • Sign in to your account at https://anna.partners.
  • Go to the App Store (or Apps marketplace) and search for litflow-ai.
  • Click Install App on your profile.
  • Open the app inside your workspace, authorize the LLM Completion permission when prompted, and start typing your research topics!

Prerequisites for Developers

If you want to clone this repository, run it locally, or contribute to the codebase, you'll need the following environment:

  • Node.js: Version 22.0.0 or higher.
  • Anna CLI: The official command-line interface.
bash 
npm install -g @anna-ai/cli

Local Development Setup

Run a local environment connected directly to your Anna account and model endpoint to test live iterations:

bash 
# 1. Login to the platform
anna-app login --host https://anna.partners

# 2. Validate the bundle schema against manifest rules
npm run validate

# 3. Start the local server
anna-app dev

Once started, open the local developer URL (usually http://localhost:5180) in your browser.

Useful package.json Commands

Here are the project commands configured in the repository:

  • npm run dev: Opens a live developer bridge to the platform capabilities.
  • npm run dev:mock: Performs UI-only smoke tests using pre-recorded mock LLM completions (no quota cost).
  • npm run validate: Statically validates the bundle setup.
  • npm run push: Pushes local changes to the platform as a working draft.
